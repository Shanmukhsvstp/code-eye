🧪 Test API Routes Documentation

Base URL: http://localhost:PORT/api

These endpoints are designed for testing and development purposes. They help verify that your API is working correctly before implementing production features.

📋 Quick Reference

Method Endpoint Purpose Auth Required
GET /test-get Basic connectivity test ❌ No

GET /api/test-get

Description

Simple endpoint to verify the API is running and responding correctly. Perfect for health checks.

Request

No parameters required.

Response

Status: 200 OK 
{
  "message": "Test successful!"
}

Example Usage

cURL:

curl http://localhost:3000/api/test-get

JavaScript (Fetch):

fetch('http://localhost:3000/api/test-get')
  .then(response => response.json())
  .then(data => console.log(data));

JavaScript (Axios):

axios.get('http://localhost:3000/api/test-get')
  .then(response => console.log(response.data));

GET /api/test-params

Description

Tests query parameter handling and validation. Verifies your API correctly reads and validates URL parameters.

Query Parameters

Parameter Type Required Default Description
password string No 12345678 Must be "12345678" to pass validation

Successful Response

Status: 200 OK 
{
  "message": "Test successful!"
}

Error Response

Status: 400 Bad Request 
{
  "error": "Invalid password"
}

Example Usage

cURL (Success):

curl "http://localhost:3000/api/test-params?password=12345678"

cURL (Failure):

curl "http://localhost:3000/api/test-params?password=wrongpass"

JavaScript (Fetch):

const params = new URLSearchParams({ password: '12345678' });
fetch(`http://localhost:3000/api/test-params?${params}`)
  .then(response => response.json())
  .then(data => console.log(data));

JavaScript (Axios):

axios.get('http://localhost:3000/api/test-params', {
  params: { password: '12345678' }
})
  .then(response => console.log(response.data));
💡 Note: If no password parameter is provided, it defaults to "12345678" and the request will succeed.

GET /api/test-token

Description

Validates that the Authorization header is present in the request. Useful for testing authentication middleware.

Headers

Header Required Description
Authorization ✅ Yes Any non-empty token value

Successful Response

Status: 200 OK 
{
  "message": "Token is valid!"
}

Error Response

Status: 401 Unauthorized 
{
  "error": "Missing token"
}

Example Usage

cURL (Success):

curl -H "Authorization: Bearer your-token-here" \
  http://localhost:3000/api/test-token

cURL (Failure):

curl http://localhost:3000/api/test-token

JavaScript (Fetch):

fetch('http://localhost:3000/api/test-token', {
  headers: {
    'Authorization': 'Bearer your-token-here'
  }
})
  .then(response => response.json())
  .then(data => console.log(data));

JavaScript (Axios):

axios.get('http://localhost:3000/api/test-token', {
  headers: {
    'Authorization': 'Bearer your-token-here'
  }
})
  .then(response => console.log(response.data));
💡 Note: This endpoint only checks if the Authorization header exists, not if the token is actually valid.

POST /api/test-login

Description

Simulates a login flow with username and password validation. Returns a mock token on success.

Request Body

Field Type Required Description
username string ✅ Yes Must be "testuser"
password string ✅ Yes Must be "Testpassword@123*"

Successful Response

Status: 200 OK 
{
  "message": "Login successful!",
  "token": "test-token"
}

Error Responses

Status: 400 Bad Request (Invalid JSON) 
{
  "error": "invalid request"
}
Status: 401 Unauthorized (Wrong credentials) 
{
  "error": "Invalid credentials"
}

Example Usage

cURL (Success):

curl -X POST http://localhost:3000/api/test-login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "Testpassword@123*"
  }'

cURL (Failure):

curl -X POST http://localhost:3000/api/test-login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "wronguser",
    "password": "wrongpass"
  }'

JavaScript (Fetch):

fetch('http://localhost:3000/api/test-login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'testuser',
    password: 'Testpassword@123*'
  })
})
  .then(response => response.json())
  .then(data => console.log(data));

JavaScript (Axios):

axios.post('http://localhost:3000/api/test-login', {
  username: 'testuser',
  password: 'Testpassword@123*'
})
  .then(response => console.log(response.data));
🔐 Test Credentials:
Username: testuser
Password: Testpassword@123*

🚀 Testing Workflow

Recommended Testing Order:

  1. GET /test-get - Verify basic connectivity
  2. GET /test-params - Test query parameter handling
  3. POST /test-login - Test request body parsing and validation
  4. GET /test-token - Test authorization headers
  5. PATCH /test-patch - Verify PATCH method support

⚠️ Important Notes

  • These endpoints are for testing and development only
  • Remove or protect these routes in production environments
  • The test token returned is not a real JWT - it's just a mock value
  • Replace localhost:3000 with your actual server address and port

🔧 Common Issues

CORS Errors

If testing from a browser and getting CORS errors, ensure your server has CORS middleware enabled:

app.Use(cors.New(cors.Config{
    AllowOrigins: "*",
    AllowMethods: "GET,POST,PATCH,DELETE,PUT",
    AllowHeaders: "Origin, Content-Type, Authorization",
}))

Connection Refused

Make sure your Go server is running:

go run main.go

404 Not Found

Verify you're using the correct base path. Check if your API uses a prefix like /api.
